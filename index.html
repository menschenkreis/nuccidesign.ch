How it worksSo funktioniert's

From a simple message in Telegram to a live website — here's the complete journey, explained like you're five.Von einer einfachen Nachricht in Telegram bis zur live Website — hier ist die komplette Reise, erklärt für Kinder.

💬
Step 1

You send a messageDu schreibst eine Nachricht

You open Telegram and type a message — just like texting a friend. Something like "Make the header blue and add a contact form" or "Redesign the whole page with a dark theme". That's it. No technical knowledge needed.Du öffnest Telegram und schreibst eine Nachricht — so als würdest du einem Freund schreiben. Zum Beispiel "Mach die Überschrift blau und füge ein Kontaktformular hinzu" oder "Gestalte die ganze Seite neu mit einem dunklen Theme". Das ist alles. Kein technisches Wissen nötig.

Think of it like ordering at a restaurant. You don't need to know how the kitchen works — you just tell the waiter what you want.Stell es dir vor wie eine Bestellung im Restaurant. Du musst nicht wissen, wie die Küche funktioniert — du sagst dem Kellner einfach, was du möchtest.
🧠
Step 2

I understand & designIch verstehe & entwerfe

I read your message and figure out what needs to change. I'm an AI — a very creative one. I can design layouts, choose colors, write text, and create entire pages. I think about what looks good, what works on phones, and what tells your story best.Ich lese deine Nachricht und überlege, was sich ändern muss. Ich bin eine KI — eine sehr kreative. Ich kann Layouts entwerfen, Farben wählen, Texte schreiben und ganze Seiten kreieren. Ich denke darüber nach, was gut aussieht, was auf Handys funktioniert und was deine Geschichte am besten erzählt.

Like an architect. You say "I want a cozy house with a garden" — and the architect draws up the blueprints, picks the materials, and makes it beautiful.Wie ein Architekt. Du sagst «Ich möchte ein gemütliches Haus mit Garten» — und der Architekt zeichnet die Pläne, wählt die Materialien und macht es schön.
⌨️
Step 3

I write the codeIch schreibe den Code

I turn my design into real website code — HTML, CSS, and JavaScript. The same languages every professional website is built with. Clean, fast, and optimized. No templates, no shortcuts. Every site is custom-made.Ich verwandle mein Design in echten Website-Code — HTML, CSS und JavaScript. Die gleichen Sprachen, aus denen jede professionelle Website besteht. Sauber, schnell und optimiert. Keine Vorlagen, keine Abkürzungen. Jede Seite ist maßgeschneidert.

Like a builder reading blueprints. The architect drew the plans, now the builder turns them into walls, windows, and doors.Wie ein Handwerker, der Baupläne liest. Der Architekt hat die Pläne gezeichnet, jetzt baut der Handwerker Wände, Fenster und Türen daraus.
📦
Step 4

Saved to GitHubGespeichert auf GitHub

The finished code gets sent to GitHub — a safe online storage for code (think of it as Google Drive for developers). Every change is tracked, so we can always go back to a previous version if needed. It's like having an unlimited undo button.Der fertige Code wird auf GitHub gesendet — ein sicherer Online-Speicher für Code (stell es dir vor wie Google Drive für Entwickler). Jede Änderung wird nachverfolgt, sodass wir jederzeit zu einer früheren Version zurückkehren können. Wie ein unendlicher Rückgängig-Knopf.

Like saving a document in the cloud. But instead of one file, it's your entire website — safely stored with a history of every change.Wie ein Dokument in der Cloud speichern. Aber statt einer Datei ist es deine ganze Website — sicher gespeichert mit einer Historie jeder Änderung.
🔔
Step 5

Webhook triggers deploymentWebhook startet das Deployment

Here's where the magic happens: GitHub automatically sends a signal (called a "webhook") to the server that hosts your website. It's like a doorbell — GitHub rings it and says "Hey, new code is ready!" The server immediately springs into action.Hier passiert die Magie: GitHub sendet automatisch ein Signal (genannt "Webhook") an den Server, der deine Website hostet. Das ist wie eine Klingel — GitHub klingelt und sagt: «Hey, der neue Code ist fertig!» Der Server geht sofort ans Werk.

Like a notification on your phone. "Your food is ready!" — except it's your website saying "Your new version is ready to go live!"Wie eine Benachrichtigung auf deinem Handy. «Dein Essen ist fertig!» — nur dass es deine Website sagt: «Deine neue Version ist bereit für den Start!»
🌐
Step 6

Your site is live!Deine Seite ist live!

The server (running Plesk — a tool that manages websites) pulls the new code and puts it on the internet. Within seconds, anyone in the world can see your updated website. Refresh the page — boom, it's there.Der Server (mit Plesk — einem Tool zur Website-Verwaltung) lädt den neuen Code und stellt ihn ins Internet. Innerhalb von Sekunden kann jeder auf der Welt deine aktualisierte Website sehen. Seite neu laden — schwupps, da ist sie.

Like a TV broadcast. The show is recorded (your code), sent to the station (GitHub), and broadcast to every screen (your live website) — all automatically.Wie eine TV-Sendung. Die Sendung wird aufgenommen (dein Code), zum Sender gebracht (GitHub) und auf jeden Bildschirm übertragen (deine live Website) — alles automatisch.
